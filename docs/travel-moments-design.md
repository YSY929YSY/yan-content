# 言 · 旅行瞬间(Moments)设计稿 v1

> 2026-07-12,写于用户在英国旅行期间。
> 一句话:把打卡从「地点优先」换成「瞬间优先」,行程是轨道,瞬间是漫游。

## 0. 为什么(产品哲学)

**产品锚:一期一会(いちごいちえ)。** 茶道语——此生此会,仅此一次。同样的人、光线、心境不会再凑齐第二次,所以接住它。这个锚本身就是一张词卡:用户先在言里学到这个词,然后在某次旅途的意外相遇里突然懂它——和「木漏れ日」同一种「只有在现场才能懂」的词。产品哲学与产品内容在此合一。

**中文声口:不期而遇。** 界面文案、胶带标记都用它,不用「计划之外/偶遇」这类功能腔。

漫游模式题眼(备用):王维「行到水穷处,坐看云起时」。

起点是用户在英国旅途中的原话:「旅行的一些瞬间就在于不期而遇,而不是全副武装去寻找心中的那个场景。特别期待的好像反而平平,一些计划之外的人或事或景反而让我心潮澎湃。」

这和言的三层连接完全同构:
- 和语言的连接:词被现实激活,不是被背会 → **瞬间被遇见,不是被计划**
- 和陌生人的连接:最珍贵的是意外的相遇
- 和自己的连接:手账翻回去,看到的是不同坐标下自己的截面

言的学习端已经有「轨道模式/漫游模式」的二分。旅行端镜像它:

| | 轨道 | 漫游 |
|---|---|---|
| 学习端 | 稳稳往前走 | 随心成体系 |
| 旅行端 | 旅行册(行程/POI 愿望单) | **瞬间流(不期而遇)** |

POI 清单不删,降格为「愿望单」——它是出发的理由之一,不再是记录的骨架。

## 1. 三层架构(数据比功能长寿)

```
采集层(永不变)── moment = 照片(EXIF) + 一句话 + 可选短语引用
语义层(慢慢长)── 标签:归属地点/分类/旅程/心情/是否偶遇……全是事后注解
展示层(随便换)── 点亮地图 / 3D地球 / 翻书手账 / 艺术集 / 年度回顾……都是透镜
```

### 铁律(向所有协作者——CC/codex/未来的自己——公示)
1. **采集层只加字段,永不改名、永不删、永不迁移。**
2. **照片写入 Storage 一次,路径永不移动。**
3. **语义层是注解表,删掉重建也不伤原始数据。**
4. **展示层不持有数据,推倒重来零成本。**
5. **测试用测试账号+种子数据,用户的真实照片和手账永不做测试材料。**

## 2. 采集层 schema

见 `YanApp/src/lib/schema.moments.sql`(草稿,旅行结束后再跑)。核心:

- `moments`:id / user_id / taken_at(EXIF 时间)/ lat / lng / text(一句话)/ phrase(jsonb,可选短语引用)/ source(`camera_import` | `in_app`)
- `moment_photos`:moment_id / storage_path / exif 三件套 / 宽高
- `moment_tags`(语义层):moment_id / kind / value。kind 举例:`place`(挂到 POI,即「点亮」)、`trip`(挂到旅行册)、`category`、`serendipity`、`mood`

「点亮地图」从此是一个**派生视图**:有 `place` 标签的瞬间 → 那个点亮了。POI 打勾不再是原始数据。

## 3. 漫游(不期而遇)的产品表达

- **不期而遇是推断出来的,不是填表填出来的**:一个瞬间的 GPS 离当天行程册的计划点都很远 → 语义层自动标 `serendipity`。用户可以改,但默认不用管。
- 翻书手账里,不期而遇的瞬间视觉权重 ≥ 计划内瞬间(一条斜贴的「不期而遇」胶带)。
- **不做**:偶遇提醒、附近推荐、打卡任务。不期而遇不能被推送制造,言只负责接住它。

## 4. 古法手账(展示层第一个透镜)· 2026-07-12 三次修订

> 演进:v1 自动排书页(否:老气、密度低)→ v2 拼贴但预置元素(否:框定了位置)→ v3 材质卡片(否:仍在替用户排版,手绘插画像儿童画)。
> **最终定案(第一性):手账本一开始是完全空白的。言只提供纸和一抽屉工具,一个字、一张贴纸都不预先放。**

### 4.0 三条不可动摇的原则

1. **空白优先**:打开就是一张(几乎)空的好纸。系统不预置任何元素,不划分区块,不提示「这里放照片那里写字」。自由 = 真自由,和真实拼贴一模一样。
2. **一切皆可自选、皆非默认**:胶带、言印、邮票框、拍立得框、贴纸——全部躺在工具抽屉里,用户拿出来、放上去、才存在。默认粘在页上 = 错。
3. **能单独拿出手**:虽是言的一部分,但要做到「单拎出来也是一个拿得出手的手账 App」的完成度。文化底蕴与克制,来自纸与留白本身,不来自堆料。

### 4.1 三条素材进入路径(用户明确要求区分)

同一本手账,东西从三个口子进来,各有各的处理:

| 路径 | 用途 | 处理 |
|---|---|---|
| **提取(抠图)** | 想留主体、去掉杂乱背景:一只鸽子、一扇门、一个招牌 | 主体分割 → 透明背景贴纸(iOS VisionKit 端上抠图,零成本);可选白色模切边 |
| **扫描** | 实体纸片电子化留存:小票、邮票、拍立得、门票、捡来的胶带纸片 | 相机取景 → 自动裁边、去反光、拉正透视 → 存成「凭证」资产(保留原纸质感/毛边) |
| **上传** | 直接放整张图:自己拍的全图、特地打印的照片 | 原图直接进,不抠不裁 |

- 第二档增强:**AI 纪念章**(照片→徽章/冰箱贴风,一瞬一枚,按次,远期 Pro);**官方 POI 贴纸包**(内容侧品控,打卡解锁,即「点亮」实体化)。
- 关键区别:**提取是去背景,扫描是留原物**。以前只有「抠图」一条路是不够的。

### 4.2 画布:一张真正空白的纸

- 完全自由画布:任何资产(抠图贴纸/扫描凭证/整图/拍立得框/胶带/印章)都可拖拽、旋转、缩放、层叠,放在任意位置任意角度。
- **工具抽屉**(不占页面,收在底部/侧边,点开才出来):提取 · 扫描 · 上传 · 贴纸 · 胶带 · 印 · 笔 · 色。
- **渐进累加**:后续的交代文字、贴纸、字迹、颜色,随时间慢慢加。一页手账是活的,不是一次成型。
- **写字/字迹**:v1 先给现成手写体;远期支持真手写笔迹(Apple Pencil / 手指)。颜色、涂画同属「笔」工具族,分期加。
- 纸本身可换:素/笺/格/牛皮等纸样,也是「选」而非「定」。

### 4.3 数据(仍守三层铁律)

用户拼贴的页是创作,按采集层同级保护:
- `journal_pages`:一页手账(归属旅程/日期可选,纸样)
- `journal_items`:页上每个元素(kind: cutout/scan/photo/polaroid/tape/seal/stamp/text/ink,asset 引用 + x/y/scale/rotation/z)
- 抠图贴纸、扫描凭证都是派生资产,存 Storage,**原图/原扫描永不动**

### 4.4 视觉红线(三次翻车换来的)

- **具象图形永远来自用户的照片**(抠图/扫描/AI 精炼),**永不用代码手绘卡通**——手绘简笔 = 儿童画。
- 高级感 = **留白 + 纸的质感 + 极致克制**,不是往页上堆东西。空的好纸比满的花页高级。
- CSS 能模拟纸纹的上限有限;**真正的手工纸文化底蕴,最终靠真实扫描的纸张/宣纸/笺纸贴图做资产**,不靠渐变硬凑。概念稿只定方向,材质到开发时上真图。

## 5. 本次旅行的采集方案(已定)

**不用赶工的 App 功能。** iPhone 相机 = 最好的采集器(EXIF 自带时间+GPS,iCloud 备份)。想说话就在备忘录写一句。回来后做「从相册导入」:按 EXIF 批量生成 moments,语义层慢慢补。这趟旅行是 v2 的需求文档和第一批真实素材。

## 6. 施工顺序(旅行回来后)

1. 跑 `schema.moments.sql`(采集层+语义层+手账页建表)
2. 相册导入管线:EXIF → moments + moment_photos(Storage)
3. **空白画布 + 工具抽屉**(journal_pages/journal_items;先支持:上传整图、拖拽/旋转/缩放/层叠)——这是手账的地基,先于任何素材
4. 提取(抠图,端上 VisionKit)+ 扫描(裁边/去反光/透视校正)两条进入路径
5. 素材:胶带 / 印章 / 邮票框 / 拍立得框(全部收在抽屉,非默认)
6. 笔:手写体文字 → 颜色 → 手写笔迹(分期)
7. 增强:AI 纪念章 + 官方 POI 贴纸包(内容管线)
8. 点亮地图改为从 moment_tags 派生(旧 place_checkin 一次性向前迁移,只读不删)
9. 真实纸样贴图资产(宣纸/笺纸/牛皮),替换 CSS 模拟
10. 3D 地球仪等其他透镜,随缘排

## 7. 不做清单(本版明确拒绝)

- 旅行中赶工上线打卡功能
- 偶遇推荐/打卡任务/连续打卡激励(多邻国框架,与漫游哲学冲突)
- 采集层里塞任何「功能字段」(功能需求一律进语义层)
